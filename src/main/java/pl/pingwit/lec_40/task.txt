1. В ваших проектах создайте UserValidator аналогии с занятием. Помимо тех методов, которые мы реализовали на занятии,
 напишите проверку входящих данных на уникальность имэйла.

2. Создайте эндпоинт для обновления продукта. В продукте возможно обновить все: название, описание, цену.
Создайте валидатор для продукта. Правила валидации: имя продукта - не пустая строка, не содержит символов $.
Описание (оно опциональное) - не пустая строка, не содержит символов $. Цена - не отрицательная и не 0.

3. Исправьте код репозиториев - открывайте Connection внутри блока try-with-resources
