Задача урока - поработать с типами данных, попрактиковаться с дизайном таблиц, неймингом колонок, скриптами

Спроектировать базу данных для хранения поездок пользователей. В системе должны существовать сущности:
    - пользователь (имя, фамилия, имэйл (уникальный), телефон (уникальный), дата регистрации, признак вип)
    - автомобиль (марка, модель, год выпуска, класс (эконом, стандарт, люкс), дата добавления в систему.
    - водитель (имя, фамилия, телефон (уникальный), номер лицензии (уникальный), дата регистрации в системе, рейтинг (по шкале от 0 до 100)
    - поездка (пользователь, машина, водитель, откуда (строка), куда (строка), время начала, время окончания, цена в рублях,
    рейтинг (от 0 до 5)
