Нужно создать базу данных заказов
в системе должны существовать пользователи: имя, фамилия, имэйл, телефон;
товары: название, описание, цена
заказы: дата создания заказа, пользователь, количество товара, статус (сформирован, оплачен, доставлен, возврат),
комментарий к заказу (комментарий опционален, может быть а может и не быть) исходим из предположения,
что в заказе может быть только ОДНО наименование товара. случай, когда наименований может быть несколько,
сложнее - это задача со звездочкой, выполняйте по желанию

изначальную схему бд спроектируйте в гугл-таблицах и пришлите для проверки
